Priceline Intern Welcome Packet
Role: Design
History of the Project Up until its inception, Priceline had been operating without a front end styleguide. This created many problems for designers, developers and, most importantly, users: designers lacked a documented design system on which to base their designs leading to inconsistencies across the site, developers found themselves repeatedly writing the same HTML and CSS without any defined naming conventions and users were presented with a site replete with inconsistencies, leading to a subpar user experience. Styleguide to the rescue! The styleguide solves these problems by: Defining a Priceline.com design system which designers can reference in a single place Defining common UI styles in code to create modularized components, which developers can quickly grab and use Creating a globally referenced base.css to reduce code bloat and make implementing global changes possible and painless Establishing scalable CSS naming conventions and promote more consistently written HTML Providing a sandbox to rapidly prototype designs and test them in code
