Priceline.com Styleguide
Role: Design & Development
Summary

The Priceline Styleguide is an ongoing project to create a living style guide and base.css for the Priceline.com desktop site that both designers and developers can reference. It’s a project I’ve been working on with Brian McCormack since my days as a wee intern at Priceline, and has taught me massive amounts about design systems, development, scalability, and workflows. Throughout the projects lifespan, I’ve been involved in many ways from conducting competitive analysis and site UI audits, to designing and coding (html and sass) modular components into the guide to helping define class naming conventions and writing documentation. Currently, the styleguide is built on Brad Frost’s Patternlab and contains a library of common UI components around across the site. These common components make up the base.css which developers can use as a project starter just like Foundation or Bootstrap. More about the entire project below.

The Problem

Priceline.com is a massive, multi-product site that has been in existence for more than a decade. During this time, Priceline expanded organically and operated without a definitive style guide; however, this created some inevitable problems for designers, developers and users. From a user perspective, the site’s lack inconsistent UI delivered a disjointed sometimes even jarring experience and undermined the site’s legitimacy. From a design perspective, there was no single reference as to what basic UI components should look like (during a button audit, my colleague Brian found at least 12 different button styles running amok on the site). Similarly, developers found themselves coding and recoding these common components repeatedly. Enter: the living styleguide! The living styleguide is an attempt to solve these issues by defining and documenting a design system that works across products.

Styleguide Goals

  • Create a consistent user experience
  • Provide a visual reference for common UI components and brand assets for designers and developers
  • Eliminate unnecessary development efforts by creating re-usable pieces of code
  • Institute CSS class naming conventions, allow for easy identifying/updating of components
  • Create one base.css for all projects to use as a starting point
  • Create and catalog a priceline font icon set to be used instead of images to improve consistency and performance

Future Styleguide

When the styleguide project started, mobile and desktop teams worked mostly independent of each other. As Priceline.com begins to explore responsive design, the line between desktop and mobile is increasingly blurred. While the two paths will remain separate for the foreseeable future, we are working on expanding our design system to include responsive patterns that will help establish conventions for these “in-between” responsive pages to ensure they align with both desktop and mobile styles and create a seamless user experience.
